# Lines_to_cover_1c

Утилита определения **покрываемых (исполняемых) строк** модулей 1С (BSL) по исходникам.

На вход — каталоги с `.bsl`, на выход — список строк, которые считаются покрываемыми
(тот же набор правил, что измеряет отладчик платформы 1С). Используется **только парсер
[bsl-parser](https://github.com/1c-syntax/bsl-parser)**: без сервера отладки, без 1С:EDT,
без установленной платформы 1С.

Предназначена для расчёта **процента покрытия**: замер (coveragebsl/Coverage41C) даёт только
*исполненные* строки, а эта утилита — *все покрываемые* строки (знаменатель). Их объединение
даёт покрытые/непокрытые строки и процент.

## Использование

```
lines-to-cover --src <каталог> [--src <каталог>...] [--base <каталог>] [--out <файл>] [--format json|generic]
```

- `--src` — каталог исходников (`.bsl`), можно указывать несколько раз;
- `--base` — база для относительных путей в выводе (по умолчанию — абсолютные);
- `--out` — файл результата (по умолчанию — stdout);
- `--format` — `json` (по умолчанию) или `generic` (genericCoverage XML для SonarQube).

Примеры:

```bash
# JSON: файл -> номера покрываемых строк
lines-to-cover --src src/cf --base . --out linesToCover.json

# genericCoverage: все строки как covered="false"
lines-to-cover --src src/cf --base . --format generic --out linesToCover.xml
```

Форматы вывода:

```jsonc
// json
{
  "src/cf/CommonModules/Модуль/Ext/Module.bsl": [3, 5, 7, 8, 12]
}
```

```xml
<!-- generic -->
<coverage version="1">
  <file path="src/cf/CommonModules/Модуль/Ext/Module.bsl">
    <lineToCover lineNumber="3" covered="false"/>
  </file>
</coverage>
```

## Установка

Готовые сборки (jlink-бандл со встроенным рантаймом — **Java устанавливать не нужно**)
публикуются в [releases](https://github.com/Segate-ekb/Lines_to_cover_1c/releases) под каждую ОС:
`lines-to-cover-windows.zip` / `lines-to-cover-linux.zip` / `lines-to-cover-macos.zip`.
Распакуйте и запускайте `bin/lines-to-cover` (`bin/lines-to-cover.bat` на Windows).

## Сборка из исходников

Нужен только JDK 21 (для сборки).

```bash
./gradlew test          # прогон тестов
./gradlew runtimeZip    # jlink-бандл: build/*.zip (свой рантайм + приложение)
./gradlew run --args="--src src/cf --format generic"
```

## Правила покрываемых строк

Покрываемой считается строка начала исполняемого оператора: присваивание, вызов
процедуры/функции, `Если`/`ИначеЕсли`, `Для`/`Пока`, `Возврат`, `Прервать`, `Продолжить`,
`Перейти`, `ВызватьИсключение`, `Выполнить`, `ДобавитьОбработчик`/`УдалитьОбработчик`; а также
строки `Цикл`, `КонецЦикла`, `КонецЕсли`, `КонецПопытки`, `КонецПроцедуры`, `КонецФункции`.
Вызовы встроенных функций языка (Строка, Число, СокрЛП, Найти, ...) отдельными строками не считаются.

Не покрываемы: `Иначе`, `Попытка`, `Тогда`, `Исключение`, объявления `Процедура`/`Функция`/`Перем`,
директивы препроцессора и области.
